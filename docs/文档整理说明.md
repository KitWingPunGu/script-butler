## ✅ 执行的操作

### **1. 创建文件夹**
```bash
# 创建 docs 文件夹（已存在）
# 创建 docs/history 文件夹
New-Item -ItemType Directory -Path "docs/history"
```

### **2. 复制文件到 docs/**
```bash
# 复制用户指南
Copy-Item QUICKSTART.md, FEATURES.md, TROUBLESHOOTING.md → docs/

# 复制开发者文档
Copy-Item ARCHITECTURE.md, DEVELOPMENT.md → docs/

# 复制项目文档
Copy-Item INDEX.md, PROJECT_SUMMARY.md → docs/
```

### **3. 复制历史文档到 docs/history/**
```bash
Copy-Item 中文化说明.md, 名称更改说明.md, 快速测试指南.md, 更新说明.md → docs/history/
```

### **4. 删除根目录原文件**
```bash
# 删除已移动的文档
Remove-Item ARCHITECTURE.md, DEVELOPMENT.md, FEATURES.md, ...
```

### **5. 更新文档索引**
- ✅ 更新 `docs/README.md` - 添加新文档分类
- ✅ 更新 `README_CN.md` - 添加文档链接部分

## 📝 维护指南

### **添加新文档时**

1. **用户文档** → 放在 `docs/` 根目录
2. **开发者文档** → 放在 `docs/` 根目录
3. **功能开发记录** → 放在 `docs/` 根目录
4. **历史记录** → 放在 `docs/history/`

### **更新文档索引**

每次添加新文档后，记得更新：
- `docs/README.md` - 添加到相应分类
- `README_CN.md` - 如果是重要文档，添加链接

### **文档命名规范**

- **英文文档**：使用大写字母开头，如 `QUICKSTART.md`
- **中文文档**：使用中文名称，如 `新功能建议.md`
- **历史文档**：保持原名称，归档到 `history/`
