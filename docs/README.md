## 📖 如何使用这些文档

### **如果你是用户**
1. 从 `README.md` 或 `README_CN.md` 开始
2. 查看 `QUICKSTART.md` 快速上手
3. 浏览 `FEATURES.md` 了解所有功能
4. 遇到问题时查看 `TROUBLESHOOTING.md`

### **如果你是开发者**
1. 阅读 `DEVELOPMENT.md` 了解开发流程
2. 查看 `ARCHITECTURE.md` 理解架构设计
3. 参考 `docs/` 目录中的技术文档
4. 查看 `CHANGELOG.md` 了解版本历史

### **如果你想贡献**
1. 阅读 `DEVELOPMENT.md` 了解开发规范
2. 查看 `docs/新功能建议.md` 了解规划中的功能
3. 参考 `docs/新功能实现说明.md` 了解实现细节
4. 提交 Pull Request 前更新相关文档

## 🔄 文档更新

### **更新原则**
- ✅ 用户文档保持简洁，面向使用场景
- ✅ 开发文档详细记录技术细节
- ✅ 内部文档记录开发过程和决策
- ✅ 所有文档保持同步更新

### **更新时机**
- 新功能开发前：更新功能建议文档
- 新功能完成后：更新实现说明文档
- 问题修复后：记录修复过程
- 版本发布时：更新 CHANGELOG.md

---

## 📝 文档编写规范

### **格式要求**
- 使用 Markdown 格式
- 使用中文编写（用户文档提供英文版本）
- 使用清晰的标题层级
- 添加代码示例和截图

### **内容要求**
- 简洁明了，避免冗余
- 提供具体示例
- 包含使用场景
- 注明注意事项

### **命名规范**
- 使用描述性的文件名
- 中文文档使用中文文件名
- 英文文档使用英文文件名
- 使用 `.md` 扩展名
